manifesto

The idea for the orangechoc theme sprang from David Scotson's Kubrick theme manifesto >>I totally subscribe to his arguments in favour of a fixed-width, 2-column layout. Orangechoc will fit a 830 pixels wide window. With higher resolutions, the Moodle content will be displayed centered, with two chocolate-color margins. This means that text lines will always be kept at a reasonable width, making text easier to read (on legibility on the Web, see http://www.maxdesign.com.au/presentation/em/ and http://www.webstyleguide.com/type/lines.html).

The orangechoc theme also adresses the often-neglected matter of text size resizing in browsers. In the two most commonly used browsers to date, i.e. MS Internet Explorer (7.0) and Firefox (2.0), if text size is set at medium (in MSIE) or normal (in Firefox), text using this theme will be displayed at almost exactly the same size, which should be the size of the text you are reading now. When text size is resized to smaller or larger, it should diminish or grow in reasonable increments (in smaller increments in Firefox than MSIE). In various parts of the site, font size is set at e.g. 85% of its normal size, to accomodate for bigger chunks of text, hopefully without adverse effects to legibility.

compatibility issues

browser compatibility

Orangechoc has been thoroughly tested with Windows XP and MS Internet Explorer (7.0) and Firefox (2.0). In Windows, those two browsers will display orangechoc almost identically, except for the customized color buttons, which will only show up in Firefox. On the Mac, I expect this theme will work fine in Firefox and maybe in Safari, and certainly not in Internet Explorer. I will welcome testing reports with full details, sent on the Moodle forums please.

Moodle compatibility

The orangechoc theme is at the time of writing compatible with Moodle versions 1.6 to 1.8 . Before reporting problems with orangechoc, please always check that you are using the latest version, available from the URL mentioned at the top of this document.

HTML doctype

In order for the orangechoc theme to display font sizes correctly in MSIE, the header.html doctype must be set as follows (and must not be changed).

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3c.org/TR/1999/REC-html401-19991224/loose.dtd">

how to use orangechoc in Moodle

dealing with blocks on the left

Because orangechoc is a 2-column layout, it will not display any Moodle blocks which have been placed in the left column on a site's or a course's welcome page. It is recommended to move any blocks from the left to the right column before selecting orangechoc. However, if you have forgotten to do so, don't panic: you can still display invisible left blocks by clicking on the Turn editing on button and then move those blocks to the right. However it is not possible to move blocks from the right to the left in orangechoc.

your company logo(s)

In orangechoc, the logo displayed on a site's home page and the logo displayed on each course's home page have been set by default to two Moodle logos, viz. logo.gif and smalllogo.gif. You can of course replace them with your own logo(s), simply by placing yours in the theme's root directory, with the same name(s). Obviously, if your logos are much bigger than the original Moodle ones, the displaying of the header sections will be altered. Also note that some colors in your logo may jar with those in the orangechoc color chart.

On the site homepage and each course homepage you can choose to display either the logo or the site/course name or both. In the file header.html, the lines to display logos are shown in blue, the lines to display site/course names are shown in red below. By removing unwanted lines you can customize your site display.

<?php if ($home) { // This is what gets printed on the home page only
// snip // ........................................................
<div class="headermain">
<img border="0" src="<?php echo "$CFG->wwwroot/theme/$CFG->theme" ?>/logo.gif">
<?php echo $heading ?>
</div>
// snip //.........................................................
<?php } else if ($heading) { // This is what gets printed on any other page with a heading
?>
<div id="header">
<div class="headermain">
<img border="0" src="<?php echo "$CFG->wwwroot/theme/$CFG->theme" ?>/logo_small.gif">
<?php echo $heading ?>

handling of the dropdown menu at the top-right corner of course pages

I personally think there are two problems with this menu. It takes up valuable space and can conflict with longish course names. It allows the student to wander aimlessly to any part of the course and end up losing track. Accordingly, I have removed the dropdown menu and replaced it with the login information (which I moved up from the footer).

If you prefer to revert to standard Moodle themes behavior, all you have to do is remove the lines in red and add the lines in blue in your header.html file.

<div class="headermain"><?php echo $heading ?></div>

background colors & graphics

customizing

The gradient effect in the navigation bar and the various blocks & topics etc. headings is provided by the file choc_gradient01.jpg.

If you want to replace the solid chocolate color in the page backgrounds by a motif of the same color, you can use choc01.jpg. In orangechoc.css, replace the line in red with the line in blue:

html {
  background: #4D1700;
html {
  background: #4D1700 url(choc01.jpg) top left repeat;

color chart

Almost all of the colors used in this theme are displayed at the bottom of this page, together with their HTML code. Other colors used in the chart include grey (e.g. for dimmed elements), red and green (e.g. for wrong and correct answers to Quiz questions).

links formatting and hovering

Normal links to be found on a Moodle site have been formatted to a pseudo underline effect. This effect is obtained by a border plus padding in order to move the line a little further down from text than the standard underline parameter would provide.

Autolinks to glossary entries have been formatted with a light background tint and an orange dotted underline effect. Upon hovering, a hand cursor appears, rather than the help cursor (arrow + question mark) usually found in Moodle themes. This has been done on purpose, to keep with the usual semantics of the hand cursor on the Web, waiting for a click on the hyperlink. The help cursor is available for a new span.help class, which you can use e.g. for hovering effects as in this example.

customizing

If you prefer to have the usual help cursor, you can edit this line in orangechoc.css.

Replace:

a.autolink.glossary:hover {
  background-color:#F1F1DF;
  border-bottom: 1px solid #FF0000;
  color:#190055!important;
  cursor: pointer;
}
with:

  cursor: help;

navigation bar etc. removed in Glossary popup window

When, inside a course resource or activity, a student clicks on an autolinked word to a Glossary entry, that Glossary entry is shown in a popup window. I do not like the fact that—in standard Moodle themes—that popup window displays a full navigation bar and at the bottom a link to the glossary itself. If the student then clicks on these links, she is taken away from the current Moodle window to another part of the site, in another window, which is misleading. The orangechoc theme will display a Glossary entry popup window without those links. In Firefox both the navigation bar and the bottom link to the Glossary are hidden, whereas in MSIE only the navigation bar is hidden.

customizing

If you want to revert to standard behavior you must modify orangechoc.css as follows. Remove those lines :

#mod-glossary-showentry p[align="right"] {
display:none;
}
#mod-glossary-showentry .entrylowersection,
#mod-glossary-showentry .navbar {
display: none;
}

heading block hidden

In Moodle there are 3 formats to present content in the central column of a course: Topics, Weekly and Social. I think that the heading block where the "name" of those formats is displayed is superfluous & takes up valuable space, so I have hidden it. If you prefer to have it displayed, just comment out or delete the following display:none line in orangechoc.css.

#course-view .headingblock {
  display:none;
}

doctoring Moodle's deprecated elements

In the course of writing orangechoc I have come across a number of deprecated elements still present in Moodle 1.5 and 1.5.2. These elements include font, b, center, nowrap, etc. As far as possible I have introduced in orangechoc workarounds to fight off the negative effects of those deprecated elements, but not always with success. When those deprecated elements eventually disappear in further versions of Moodle I will try to keep track and modify orangechoc accordingly.

To see the example of the nowrap problem: http://moodle.org/mod/forum/discuss.php?d=25147#142051

make your theme do cron for you

At the end of the header.html file you can add a bit of code which will automatically do cron for you. Refer to this discussion for help. If you want to do this, simply add these lines right at the end of header.html:

<?php
if(file_exists($CFG->dataroot."/cron.time") && time()-filemtime($CFG->dataroot."/cron.time")>3600){
$ch = curl_init($CFG->wwwroot."/admin/cron.php");
$fp = fopen($CFG->dataroot."/cron.time", "w");
curl_setopt($ch, CURLOPT_FILE, $fp);
curl_setopt($ch, CURLOPT_HEADER, 0);
curl_exec($ch);
curl_close($ch);
fclose($fp);
}
?>
Note that this little script will have no effect whatever on a local installation of Moodle.

by way of conclusion

The orangechoc.css file should not be considered as a good example of stylesheet for Moodle as it does not follow the pattern of breaking up the styles into several stylesheets.

I shall be very grateful to Moodle users who will test the orangechoc theme and report any problems. In order to make this a fruitful dialogue, please always state the following info. when reporting.

Moodle version; platform (Windows, Mac, Linux); browser name and version number.

Please post all reports & questions on the Moodle themes forum.

However, I will not have the time to answer queries regarding your own customization of this theme. Any discussion will deal with the latest available version of orangechoc.

You are totally free to use this orangechoc theme as is, to alter it, copy it, share it, etc. I would appreciate if you would leave the identifying info at the beginning of the orangechoc.css file (but you won't burn in Hell if you don't). I would also appreciate to hear of Moodle users who would adopt this theme for a distribution site (with reference to the site's URL).
